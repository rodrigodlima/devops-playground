Trivy Live Demo

SETUP

1 · Download DB

Before we start, Trivy needs its vulnerability database. Let's pull it once.

trivy image --download-db-only
IMAGE SCAN

2 · Scan regular Alpine image

Here's a Node app built on Alpine. Trivy finds HIGH and CRITICAL CVEs — there's quite a list.

trivy image --ignore-status fixed --severity HIGH,CRITICAL hello-nodejs24-alpine:latest
IMAGE SCAN

3 · Scan Distroless image

Same app, distroless base. No shell, no package manager — attack surface drops dramatically.

trivy image --ignore-status fixed --severity HIGH,CRITICAL hello-nodejs24-distroless:latest
⚠ Not in the presentation — but worth knowing:

Distroless images contain only your application and its runtime dependencies — no shell (bash/sh), no package manager (apt/apk), no system tools. They are maintained by Google and are available for Node, Python, Java, Go, and others.

From a security standpoint this matters a lot: if an attacker exploits a vulnerability in your app, they land in a container with almost nothing to work with — no way to download tools, run scripts, or pivot easily. Combined with the smaller image size, it significantly reduces your attack surface.

IMAGE SCAN

4 · Compare image sizes

Distroless is also smaller. Less bloat, less exposure.

docker image ls --format "table {{.Repository}}\t{{.Tag}}\t{{.ID}}\t{{.CreatedSince}}\t{{.Size}}" | grep hello
IMAGE SCAN

5 · Show only unfixed CVEs

Not every CVE has a patch yet. With --ignore-status fixed we focus only on what currently has no fix — these are the ones that actually keep you awake.

trivy image --ignore-status fixed --severity HIGH,CRITICAL python:3.13.12-slim-trixie
SECRETS

6 · Secret scanning – filesystem

Trivy also finds secrets committed to the repo. Watch it catch API keys, tokens, and more in seconds.

trivy fs --scanners secret .
SECRETS

7 · Custom secret rules

Generic patterns won't catch something like a plain DB_PASSWORD. With a custom config we define our own regex rules to cover internal secrets.

trivy fs --scanners secret --secret-config ./trivy-secret.yaml ../python-code-example/ 2>&1
SBOM

8 · Generate SBOM & scan it

We can export a full Software Bill of Materials in CycloneDX format, then scan the SBOM directly — useful for audits or offline pipelines.

trivy image --format cyclonedx --output /tmp/python-sbom.json python:3.8-slim
trivy sbom --ignore-status fixed --severity HIGH,CRITICAL /tmp/python-sbom.json
CI/CD

9 · Break the pipeline on CRITICAL

In CI, --exit-code 1 makes Trivy fail the build when criticals are found — the pipeline stops, the team is forced to act.

trivy image --exit-code 1 --severity CRITICAL python:3.8-slim; echo "Exit code: $?"
trivy image --severity CRITICAL python:3.8-slim; echo "Exit code: $?"

Without the flag, Trivy reports but exits 0 — pipeline keeps going. Your choice.

KUBERNETES

10 · Cluster misconfiguration scan

Finally, Trivy can scan a live cluster for misconfigurations — privileged pods, missing network policies, and more.

trivy k8s --scanners misconfig --report all minikube
